1.หน้าที่ของ ROS Master 
     คือ master ทำหน้าที่เป็น server สำหรับให้ node หลายๆตัวเชื่อมต่อกัน และสามารถส่ง message หากันได้
2.อธิบายทำางาน
     2.1 ROS Topics คือ ข้อความจะถูกส่งผ่านระบบการขนส่งที่มีการ  publish / subscribe  โหนดที่ส่งข้อความจะเรียกว่า publishing ชื่อที่ระบุใน Topic จะใช้เพื่อระบุเนื้อหาของข้อความ โหนดที่สนใจในข้อมูลบางประเภทจะทำการ subscribe หัวข้อที่ต้องการ อาจมีการ publish / subscribe หลายรายพร้อมกันสำหรับหัวข้อเดียวและโหนดเดียวอาจ publish / subscribe ได้หลาย Topics โดยทั่วไป publish / subscribe จะไม่ได้ตระหนักถึงการมีอยู่ของกันและกัน หรือคือการแยกกันผลิตข้อมูลตามหน้าที่ของตนเอง  
     2.2 ROS Service คือ  รูปแบบการ publish / subscribe เป็นแนวคิดในการสื่อสารที่ยืดหยุ่นมาก แต่ไม่เหมาะกับการขนส่งทาง    เดียวอย่างต่อเนื่อง จะต้องมีการใช้บริการ Request / reply กำหนดการส่งข้อความ โดยสร้างข้อความหนึ่งสำหรับการร้องขอและอีกหนึ่งสำหรับการตอบกลับ เพื่อส่งข้อความที่ร้องขอไปยังโหนดที่ร้องขอ
3.ความแตกต่างของ Topic กับ message 
      คือ message สามรถมีnode แต่ละตัวจะคุยกันผ่าน message, โดย message ก็อาจจะเป็นชนิดของตัวแปร เช่น integer, floating point, boolean หรืออาจจะเอามาประกอบกันเป็น structure เช่นประกอบเป็น array หรือประกอบรวม int, float, string เข้าด้วยกันก็ได้ แต่ Topic มีpublisher node จะ publish message ของตัวเองขึ้นไปที่ topic แล้วจะมี subscriber node มารอรับ message ที่ส่งมาทาง topic ที่มีชื่อเดียวกัน  
4.ความแตกต่างในเชิงการใช้งานของ ROS Topic กับ ROS Service 
      คือ ROS Topic มีpublisher node จะ publish message ของตัวเองขึ้นไปที่ topic แล้วจะมี subscriber node มารอรับ message ที่ส่งมาทาง topic ที่มีชื่อเดียวกัน  ส่วนROS Service จะต้องมีการใช้บริการ Request / reply กำหนดการส่งข้อความ โดยสร้างข้อความหนึ่งสำหรับการร้องขอและอีกหนึ่งสำหรับการตอบกลับ เพื่อส่งข้อความที่ร้องขอไปยังโหนดที่ร้องขอ
